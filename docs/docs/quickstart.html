Quickstart — Palace-X

Get up and running in 5 minutes. Installation, basic HNSW usage, RaBitQ compression, and combined pipeline.

Step 1

Prerequisites

Before you start, make sure you have the following installed:

  • Rust 1.75 or later (rustc --version)
  • macOS or Linux operating system
  • Optional: Apple Silicon (M1/M2/M3) for NEON SIMD and Metal GPU acceleration

Install Rust: Visit rustup.rs and follow the installation guide.

Step 2

Installation

Add palace-x to your project in two ways:

Option A: Cargo.toml

$ cargo add palace-x

Option B: Manual Cargo.toml edit

# Cargo.toml
[dependencies]
palace-x = "0.2"

Option C: Clone from GitHub

$ git clone https://github.com/Neirotunes/palace-x.git
$ cd palace-x
$ cargo build --release
Step 3

Basic HNSW Usage

Build a simple index and search:

// src/main.rs
use palace_x::HnswIndex;

fn main() {
  // Create index with 128-dim vectors
  let mut index = HnswIndex::new(128, 16);

  // Insert vectors
  for i in 0..10_000 {
    let vec = vec![i as f32; 128];
    index.insert(vec, i);
  }

  // Publish snapshot (persist to disk)
  index.publish_snapshot("index.bin");

  // Search — 99.9% recall, sub-ms latency
  let query = vec![0.5; 128];
  let results = index.search(&query, 10);

  for (label, distance) in results {
    println!("Label: {}, Distance: {}", label, distance);
  }
}

Key methods:
HnswIndex::new(dim, m) — Create index with dimension and max degree
insert(vec, label) — Add a vector with metadata label
publish_snapshot(path) — Persist index to disk
search(&query, k) — Find k nearest neighbors

Step 4

RaBitQ Compressed Search

Use 4-bit quantization for 8× memory compression:

// RaBitQ: Randomized Binary Quantization
use palace_x::RaBitQIndex;

let mut rabitq = RaBitQIndex::new(384, 4);

// Encode vectors to 4-bit representation
for i in 0..100_000 {
  let vec = get_embedding(i);
  let encoded = rabitq.encode_multibit(&vec, 4);
  rabitq.insert(encoded, i);
}

// Fast top-k search with compressed distances
let query = get_embedding(0);
let candidates = vec![/* ... */];
let topk = rabitq.rabitq_topk(&query, &candidates, 10);

Key methods:
RaBitQIndex::new(dim, planes) — Create with 1, 2, or 4 bit-planes
encode_multibit(&vec, bits) — Quantize vector to bit-planes
rabitq_topk(&query, &candidates, k) — Find top-k with 4-bit distances

Step 5

Combined Pipeline: HNSW + RaBitQ

Production-grade: HNSW graph descent + RaBitQ compressed scoring + topological reranking:

// Full pipeline: 99.9% recall, 5700 QPS
use palace_x::{HnswRaBitQ, Config};

let config = Config {
  dim: 384,
  hnsw_m: 16,
  hnsw_ef_c: 200,
  rabitq_planes: 4,
  metric: "l2",
};

let mut index = HnswRaBitQ::new(config);

// Load SIFT-1M dataset
let (base, queries) = load_sift1m("data/sift1m.bin");

// Batch insert — 10K vec/sec
for (i, vec) in base.iter().enumerate() {
  index.insert(vec.clone(), i as u32);
}

index.publish_snapshot("sift1m.index");

// Search — 99.9% R@10, ~170μs/query
for query in queries.iter() {
  let results = index.search(query, 10);
  validate_recall(&results, query);
}

Config fields:
dim — Vector dimensionality
hnsw_m — Max degree (16–32 typical)
hnsw_ef_c — Construction effort factor (200–400)
rabitq_planes — 1, 2, or 4 bit-planes
metric — "l2" or "cosine"
